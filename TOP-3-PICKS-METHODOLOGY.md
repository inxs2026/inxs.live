# TOP 3 PICKS METHODOLOGY
## Carlo's Racing Analysis Process

**Purpose:** Objective, repeatable system for picking top 3 contenders in any race.
**Last Updated:** April 10, 2026

**Related:** See [[racing/methodology/CLAIMING-PROSPECTS-CRITERIA.md]] for horses to claim (buy), not bet.

---

## THE METHODOLOGY (What I Follow)

### 1. CALCULATE BEYER AVERAGES (Last 3 races)
• Do this for EVERY pick
• Provides objective baseline to compare horses
• Format: "Avg Beyer (last 3): XX.X"

### 2. RECENT FORM ANALYSIS (Last 3-5 races)
• Look at finishes (close 2nds better than distant 4ths)
• Competitiveness (lost by nose > lost by 10 lengths)
• Consistency (2-2-3 better than 1-9-9)
• Trajectory (improving 3-2-1 vs declining 1-4-8)

### 3. CLASS ANALYSIS
• Dropping class = positive
• Proven at this level = confidence
• Rising too fast = concern
• Compare best figures vs today's field

### 4. TRAINER/JOCKEY ANGLES
• Trainer % stats from DRF ("Joseph Jr 25% with 2nd starters")
• Elite jockey switches = positive
• Recent claims show confidence

### 5. PACE SCENARIO
• Lone speed = advantage
• Slow pace suits closers
• Post position matters for running style

### 6. SURFACE/DISTANCE FIT
• Breeding angles
• Past performance at distance
• Surface switches (turf ↔ dirt ↔ synthetic)

## HOW I RANK THE TOP 3

### #1 PICK - Must have MOST of:
• Best/competitive Beyers
• Recent positive form (wins, close 2nds)
• Class advantage or proven at level
• Strong trainer/jockey angle
• Favorable pace OR sharp works

### #2 PICK - Strong but missing 1-2 elements
• Similar figures but less consistency
• Good class but no recent win
• Strong connections with one question mark

### #3 PICK - Viable with more questions
• Upside/improving
• Proven but inconsistent
• Class edge but form concerns

## THE ACTUAL PROCESS (Step-by-step)

### FOR EACH RACE:

1. **Read DRF** past performances for the field
2. **Calculate Beyer average** (last 3) for all contenders
3. **Note recent finishes** and form
4. **Check class level** vs today
5. **Look for strong trainer/jockey angles**
6. **Consider pace** (who has speed, who closes)
7. **Rank horses** by who has MOST positive factors
**71⁄2. Read comment lines** and scan for useful info (excuses, quit signs, mechanical issues)
   • Trouble/excuses: "checked", "bumped", "off slowly", "blocked", "wide trip"
   • Quit signs: "turned away", "gave way", "stopped", "no response"
   • Mechanical issues: "wrong lead", "lugged in/out", "bore in/out"
   • Positives: "finished well", "strong finish", "rallied", "drew clear"
   • Example: "checked 1/4, rallied late" = legitimate excuse for bad finish
   • Example: "turned away final furlong" = concern despite decent finish
8. **Write 2-3 sentence explanation** per pick **WITH POST POSITION**
   • Format: `**1. #3 HORSE NAME** - Avg Beyer (last 3): XX.X`
   • Post position (#) is MANDATORY - prevents data scrambling errors
9. **VERIFY DATA INTEGRITY** before creating PDF
   • Cross-check: Does horse name match this post position in DRF?
   • Cross-check: Do Beyers match this specific horse's past performances?
   • Cross-check: Does trainer match this horse?
   • **STOP if ANY mismatch** - fix before proceeding

## CRITICAL RULES

✅ **Always calculate Beyers** - No pick without Beyer average
✅ **Explain in 2-3 sentences** - If you can't, rethink the pick
✅ **Objective over gut** - Numbers + form, not hunches
✅ **Class matters** - A $12.5K horse vs $35K horses has uphill battle
✅ **Recent form > old form** - Last 3 races matter most

❌ **Don't pick on name alone** - Need data to support
❌ **Don't ignore class** - Can't wish away talent gaps
❌ **Don't skip the math** - Beyer averages are mandatory
❌ **Don't report missing/wrong without re-reading first** - If you think something is missing or wrong, re-read the PDF carefully before telling Carlo. The pattern of "finding it after being pushed" means you didn't read thoroughly enough. Re-verify before reporting any issue.

## BEYER INTERPRETATION (What Beyers Tell You — and What They Don't)

**Key principle from Carlo (April 14, 2026):** *"Beyers tell a good story but not the whole story all the time."*

### What Beyers Tell You:
- **High Beyer + fast pace** = legitimate speed, wired the race
- **High Beyer + slow pace** = inflated by easy setup (don't overvalue)
- **Low Beyer + closing run** = might be better than number suggests if trip was bad
- **Declining Beyers** = horse in decline or health issue
- **Sporadic Beyers** = inconsistent, trip/surface dependent

### What Beyers DON'T Tell You:
- Trip trouble (blocked, wide, checked)
- Equipment changes (blinks, tongue tab, etc.)
- Surface/distance shift (synth to turf, 6F to 1 mile)
- Trainer motivation (claiming vs developing)
- Weight changes or jockey switch
- Mental issues (shy, barn sour, equipment problems)

### Reading Beyer Trend Over Last 3-4 Starts:
**CRITICAL:** Flat average hides trajectory. Two horses with 43 avg are different:
- Horse A: 30 → 40 → 50 (improving trend)
- Horse B: 50 → 45 → 40 (declining trend)

Always note the direction of the trend, not just the average.

### Practical Application:
1. Note Beyer avg as foundation
2. Read comment lines for trip/excuses
3. Check if Beyers came at same/different surface/distance
4. Look for improving vs declining trend over last 3-4 starts
5. Apply pace/class/trainer angles on top

## WHEN TO DEVIATE

**First-time starters:** No Beyers available. Use:
- Breeding (sire/dam)
- Works (bullet works = positive)
- Connections (hot trainer%)
- Patience (debut after time suggests readiness)

**Lightly raced horses:** Fewer than 3 races. Use:
- Available Beyers + breeding
- Class of prior races
- Works since last race

**Long layoffs:** No recent Beyers. Use:
- Best career Beyers
- Works pattern (frequent/sharp = ready)
- Trainer % with layoffs

## VERIFICATION PROTOCOL (MANDATORY - DO NOT SKIP)

### STEP A - GET TRACKDATA FIRST (Before Writing ANY Picks)
1. Call `get_race_details(race_key="GP-YYYYMMDD-N")` for EACH race
2. Write down ALL post positions and horse names from TrackData
3. This is your ground truth - horses are matched to PPs here, not in the DRF
4. **DO NOT** write a single pick until you have this list

### STEP B - MATCH DRF TO TRACKDATA
1. For each horse in TrackData, find their DRF entry
2. Extract Beyer figures, trainer, recent works from DRF
3. Assign the PP# from TrackData (NOT from DRF - DRF PP format can be ambiguous)
4. Verify: Does this horse's name appear in TrackData for this PP#?

### STEP C - APPLY SCRATCHES (Before Picking)
1. Call `get_scratches(track="GP", start_date, end_date)` for today
2. Remove ALL scratched horses from consideration
3. Verify: Is the scratched horse in your picks? If YES → remove it
4. A scratched horse is NEVER a pick, no matter how good the data looks

### STEP D - CROSS-CHECK EVERY PICK (Before PDF Generation)
For each pick in your analysis:
- [ ] Horse name appears in TrackData entries for this race
- [ ] PP# matches what TrackData shows for this horse
- [ ] Horse is NOT in today's scratches list
- [ ] Beyer figures match THIS horse's DRF entry, not another horse
- [ ] Trainer matches THIS horse's DRF entry

### STEP E - FINAL RE-VERIFY (Before Hitting Send)
After PDF is generated but BEFORE emailing:
1. Open the PDF - does it look right?
2. Check 3 specific horses against TrackData: PP#, name, odds
3. If anything looks wrong → STOP, go back, fix, re-verify

**RULE: If you can't verify it, don't write it. Better late with correct picks than on-time with wrong ones.**
